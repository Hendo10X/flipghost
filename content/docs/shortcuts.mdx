The workshop is built to be driven from the keyboard with one hand while the
other draws. The Hotkeys button in the header shows this same list, and lets you
change it.

## The six that matter

If you learn nothing else, learn these. They cover the loop you repeat a few
hundred times an hour.

| Action | Key |
| --- | --- |
| Brush | <kbd>B</kbd> |
| Eraser | <kbd>E</kbd> |
| Add frame | <kbd>N</kbd> |
| Previous frame | <kbd>←</kbd> |
| Next frame | <kbd>→</kbd> |
| Play / pause | <kbd>Space</kbd> |

Draw, add, draw, arrow back to check, space to watch it. That is the job.

## Full list

These can all be rebound.

| Action | Default |
| --- | --- |
| Select tool | <kbd>V</kbd> |
| Brush tool | <kbd>B</kbd> |
| Eraser tool | <kbd>E</kbd> |
| Play / pause | <kbd>Space</kbd> |
| Previous frame | <kbd>←</kbd> |
| Next frame | <kbd>→</kbd> |
| Toggle onion skin | <kbd>O</kbd> |
| Add frame | <kbd>N</kbd> |

## Reserved

These are fixed and cannot be rebound, because they mean the same thing in
every application you use and it would be rude to change them.

| Action | Key |
| --- | --- |
| Undo | <kbd>Ctrl</kbd> <kbd>Z</kbd> |
| Redo | <kbd>Ctrl</kbd> <kbd>Shift</kbd> <kbd>Z</kbd> |
| Zoom in | <kbd>Ctrl</kbd> <kbd>+</kbd> |
| Zoom out | <kbd>Ctrl</kbd> <kbd>-</kbd> |
| Fit to viewport | <kbd>Ctrl</kbd> <kbd>0</kbd> |
| Delete selection | <kbd>Del</kbd> |
| Deselect | <kbd>Esc</kbd> |

## Rebinding

Open the Hotkeys menu, click the key next to an action, and press the key you
want. Letters, digits, <kbd>Space</kbd>, and the arrow keys are all accepted.
Combinations involving <kbd>Ctrl</kbd>, <kbd>Alt</kbd>, or <kbd>Cmd</kbd> are
not, because those belong to the reserved list above.

If you bind a key that another action already owns, the two swap rather than
one being left with nothing. Binding <kbd>B</kbd> to the eraser when the brush
has it gives the brush whatever the eraser was using.

There is a reset button in the menu to put everything back.

Your bindings are stored in your browser, not in your account. They apply to
every animation you open on that device, and they do not follow you to another
one.

<Callout>
  Shortcuts are ignored while you are typing in a text field, so renaming an
  animation will not scatter frames around.
</Callout>
